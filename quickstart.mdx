title: "Quick Start: First Project"
description: "Create a project, prepare AI interviews, add recordings, and generate a shareable report in minutes."

Use this quick start to create your first project report.

## Outcome
- A project with interviews and transcripts
- Single reports for each interview
- A project report ready to share

## Steps
<Steps>
  <Step title="Create a project">
    Click **Create Project** in your workspace and name the study.

    <Frame caption="Create Project modal">
      <img src="/images/quickstart-create-project-light.png" alt="Create Project modal showing project name input and file upload area" className="block dark:hidden" />
      <img src="/images/quickstart-create-project-dark.png" alt="Create Project modal showing project name input and file upload area" className="hidden dark:block" />
    </Frame>

    {/* TODO: Add screenshot

      Description: Create Project modal with project name field, optional file upload area, and Create Project button.
      Context: Step 1 showing where users create a new project.
      Dark variant: images/quickstart-create-project-dark.png
    */}
  </Step>

  <Step title="Add research context and interview guide">
    Define your **Research Goals** and **Interview Guide** in the same setup step.

    Use the **Research Assistant** to generate both through guided Q&A, or fill both fields manually.

    <Frame caption="Research Assistant section">
      <img src="/images/quickstart-research-context-light.png" alt="Research Assistant section showing Research Goals and Interview Guide cards" className="block dark:hidden" />
      <img src="/images/quickstart-research-context-dark.png" alt="Research Assistant section showing Research Goals and Interview Guide cards" className="hidden dark:block" />
    </Frame>

    {/* TODO: Add screenshot
      File: images/quickstart-research-context-light.png
      Description: Project page section with Research Goals and Interview Guide cards visible in the same viewport.
      Context: Step 2 showing where users prepare context and interview guide.
      Dark variant: images/quickstart-research-context-dark.png
    */}

    <Tip>
      Clear goals and a clear guide lead to sharper interviews and better reports.
    </Tip>
  </Step>

  <Step title="Create AI interview links and send them">
    Open **AI Interviewer** in the project, create links, and send one link to each respondent.

    **Learn more:** [AI Interviewer](/projects/ai-interviewer)

    {/* TODO: Add screenshot
      File: images/quickstart-ai-interviewer-links-light.png
      Description: AI Interviewer section with Create Links action and list of generated links ready to copy and send.
      Context: Step 3 showing link creation and respondent distribution.
      Dark variant: images/quickstart-ai-interviewer-links-dark.png
    */}
  </Step>

  <Step title="Upload previously recorded interviews">
    Add past interviews from outside Aseed (audio, video, or text) in **Project Records**.

    <Frame caption="Upload Records button">
      <img src="/images/quickstart-upload-records-light.png" alt="Project Records section with Upload Records button" className="block dark:hidden" />
      <img src="/images/quickstart-upload-records-dark.png" alt="Project Records section with Upload Records button" className="hidden dark:block" />
    </Frame>

    {/* TODO: Add screenshot
      File: images/quickstart-upload-records-light.png
      Description: Project Records section with Upload Records button and records area.
      Context: Step 4 showing where users add previously recorded interviews.
      Dark variant: images/quickstart-upload-records-dark.png
    */}
  </Step>

  <Step title="Transcribe and review (if needed)">
    Generate transcripts for audio/video files. Review and edit transcripts as needed.

    **Learn more:** [Transcribe interviews](/interviews/transcribe) and [Edit transcripts](/interviews/edit)
  </Step>

  <Step title="Generate single reports">
    Click **Generate Reports** and start with **Transcript** and **Q&A** for each interview. See [Report types](/reports/report-types) to understand what each report contains.

    <Note>
      These reports become the base material for your project report.
    </Note>
  </Step>

  <Step title="Create a project report">
    Click **Add Report**, choose a preset or custom structure, and generate the report.

    <Frame caption="Report templates">
      <img src="/images/quickstart-report-templates-light.png" alt="Report setup page showing preset templates" className="block dark:hidden" />
      <img src="/images/quickstart-report-templates-dark.png" alt="Report setup page showing preset templates" className="hidden dark:block" />
    </Frame>

    {/* TODO: Add screenshot
      File: images/quickstart-report-templates-light.png
      Description: Project report setup page with preset templates and generate action.
      Context: Step 7 showing project report creation from templates.
      Dark variant: images/quickstart-report-templates-dark.png
    */}

    <Tip>
      After you have the reports you need, use **Ask Project** in the [Research Assistant](/projects/context): select your project and ask questions to clarify or interpret findings. Answers are grounded in your project documents with citations.
    </Tip>
  </Step>

  <Step title="Share or export">
    Share a public link or export the report.
  </Step>
</Steps>
